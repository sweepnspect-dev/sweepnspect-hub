Data Pipeline

How applications flow

Visitor fills form → POST /api/founding → Cloudflare KV
   KV stored    → Event created → GET /api/events → HQ Dashboard
Visitor opens chat → Tawk.to → Webhook → Cloudflare KV
Email to contact@ → IMAP Poller → Email Router → Auto-classify

Email router classifies by bracket prefix: [FOUNDING] → auto-create subscriber, [CONTACT] → general, [BUG] → auto-create ticket.

Worker API Endpoints
Method Path Description
POST /api/founding Submit founding application — stores in KV, creates HQ event
GET /api/founding List all founding applications with status
POST /api/webhooks/tawk Tawk.to webhook receiver — chat:start, chat:end, ticket:create
GET /api/events Poll unacknowledged events (HQ dashboard consumes this)
POST /api/events/ack Acknowledge events after processing
GET /health Health check — returns status, service name, timestamp
